**Required sections per D-14:**

**Section 1: Setup**
- Required env vars: `MAINLAYER_API_KEY` (authenticates API/MCP requests), `MAINLAYER_WALLET_PASSPHRASE` (unlocks wallet for signing without interactive prompt)
- Optional env vars: `MAINLAYER_API_URL` (override API base URL, default: `https://api.mainlayer.io`), `MAINLAYER_SOLANA_NETWORK` (default: `solana:mainnet`)
- `--json` flag: all commands support it; also auto-enabled when stdout is not a TTY
- `--api-key <key>` flag: per-command auth override
- Exit codes table: 0=success, 1=general error, 2=auth error, 3=not found, 4=validation error, 5=already exists

**Section 2: Vendor Quick-Start**
Step-by-step for an AI agent acting as a vendor:
1. `mainlayer auth register --email ... --password ... --json`
2. `mainlayer auth api-key create --label "my-agent" --json` — save the key to `MAINLAYER_API_KEY`
3. `mainlayer wallet create --json` — creates encrypted wallet
4. `mainlayer resource create --slug my-api --description "..." --price 0.01 --fee-model per_call --type api --json`
5. `mainlayer resource plan create --resource-id <id> --name "Basic" --price 9.99 --interval monthly --json`
6. `mainlayer webhook update --url https://... --json`

**Section 3: Buyer Quick-Start**
Step-by-step for an AI agent acting as a buyer:
1. `mainlayer auth register --email ... --password ... --json`
2. `mainlayer wallet create --json` — fund with SOL + USDC
3. `mainlayer discover --json` — browse available resources
4. `mainlayer buy <resource-id> --json` — X402 on-chain payment
5. `mainlayer entitlements --json` — verify access

**Section 4: Full Command Reference**
Document every command group with subcommands and key flags:
- `mainlayer auth` — register, login, logout, status, api-key (create/list/revoke)
- `mainlayer wallet` — create, import, address, balance, export
- `mainlayer config` — get, set
- `mainlayer resource` — create, list, get, update, delete, plan (create/list/update/delete), quota (set/get/delete)
- `mainlayer coupon` — create, list, delete
- `mainlayer webhook` — update, logs, retry, rotate-secret
- `mainlayer earnings` — (--resource-id, --period, --from, --to)
- `mainlayer metrics` — (--resource-id)
- `mainlayer discover` — (--query, --type, --page, --limit)
- `mainlayer buy` — <resource-id>
- `mainlayer entitlements` — (--resource-id)
- `mainlayer subscribe` — approve, pause, resume, cancel, list, get
- `mainlayer invoices` — (--resource-id)
- `mainlayer refund` — request <payment-id>
- `mainlayer dispute` — create, list
- `mainlayer setup` — (--force, --json)

For each command include: brief description, key flags, expected JSON output shape (e.g., `{ id, slug, status }`).

- Use exit codes to determine success/failure programmatically
- Claude Desktop users: MCP requires manual setup via Settings > Connectors (auto-config not supported)
