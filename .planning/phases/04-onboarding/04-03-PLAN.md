**1. Create `src/cli/setup.ts`:**

```typescript
import { Command } from 'commander';
import ora from 'ora';
import { configurePlatforms, PLATFORMS } from '../postinstall/platforms.js';
import { generateSkillsMd, SKILLS_FILENAME } from '../postinstall/skills-template.js';
import { writeFileSync, mkdirSync, existsSync } from 'node:fs';
import { join, dirname } from 'node:path';
import { homedir } from 'node:os';
import { formatOutput, printSuccess, printError } from '../utils/output.js';
```

Export `setupCommand()` returning a Commander `Command` with:
- Name: `setup`
- Description: `Detect AI platforms and configure Mainlayer MCP server`
- Options: `--force` (re-write MCP entries even if already configured), `--json` (machine-readable output)
- Action handler (async):

Action handler logic:
1. Determine json mode: `opts.json || !process.stdout.isTTY`
2. If human mode, create ora spinner: `ora('Detecting AI platforms...')`
3. Call `configurePlatforms({ force: opts.force ?? false })`
4. For each result where `configured === true`, drop skills.md:
   - Get platform descriptor from PLATFORMS by matching `name`
   - Resolve skillsDir using `homedir()`
   - Ensure directory exists with `mkdirSync(dir, { recursive: true })`
   - Write `generateSkillsMd()` to `join(dir, SKILLS_FILENAME)`
   - Set `result.skillsDropped = true`
   - Wrap in try/catch — if skills write fails, log error but don't fail
5. If human mode, stop spinner, then for each platform result:
   - Configured: green checkmark + name
   - Skipped (Claude Desktop): yellow info + name + skipReason
   - Not detected: gray dash + name
   - Error: red X + name + error
   - Print final summary line: `printSuccess('Configured MCP for: X, Y')`
6. If json mode: `formatOutput({ platforms: results }, { json: true })`
